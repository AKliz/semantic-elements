Semantic Elements

What is semantics in HTML?

In the past html coders would write documents using basic non-descriptive HTML tags for example div and span. These tags are bad but they don't tell the computer much about the code.

With the introduction of HTML5 came the idea that we should also use HTML5 semantics which is pretty much alternatives to the div and span blank HTML tags. These new suggested tags include header, nav, section, article and footer. These are stronger HTML tags because they give meaning to the content they hold. This is especially important for when computers are reading your code for Assesibilty purposes and for your google rankings for when bots read your pages.

Why use semantic tags?

Primarily to enhace:

  • Accessibility - help assistive technologies read and interpret your webpage
  • Searchability - help computers make sense of your content
  • Interoperability - help other programmers understand the structure of your webpage

Want to learn more?

Don't worry about memorizing every html tag right out of the gate. It takes time. Make sure you bookmark reliable resources you can always reference back to when coding a new document.
